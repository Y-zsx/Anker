# AI 智能耳机 Web 仿真系统 — 市场调研与技术要点

## 一、市面竞品调研

### 1.1 主要产品对标

| 产品 | 厂商 | 核心功能 | APP界面特点 | 参考意义 |

### 1.2 共同的产品链路特征

所有竞品都遵循相同的 **"设备→采集→AI→结果"** 四阶段闭环：

```
[设备连接] → [语音采集] → [AI处理] → [结果展示]
   ↓              ↓            ↓           ↓
 蓝牙配对     录音控制     STT转写     转写文本
 电量显示     波形动画     翻译       翻译对照
 信号强度     降噪参数     摘要       摘要卡片
                         置信度      导出分享
```

### 1.3 交互设计共性

1. **设备面板常驻顶部** — 电量、信号、固件版本以小卡片形式固定显示
2. **场景模式切换** — 会议/学习/办公模式以 Tab 或下拉菜单切换
3. **录音控制居中突出** — 大圆形录音按钮位于界面中央，暂停/停止在两侧
4. **结果以卡片气泡呈现** — 转写文本按说话人分段，翻译以对照卡片展示
5. **进度动画反馈** — AI处理阶段显示加载动画+进度条+置信度动态更新
6. **异常友好引导** — 断连弹窗带"重新连接"按钮，权限不足带引导说明

## 二、系统架构设计

### 2.1 三层架构

```
┌─────────────────────────────────────────────┐
│              UI 展示层 (Web)                 │
│  DevicePanel | AudioCapture | AIResult |     │
│  PerformanceReport | ScenarioSwitch         │
├─────────────────────────────────────────────┤
│            业务逻辑层 (JS)                    │
│  设备模拟层 | 音频模拟层 | AI处理模拟层       │
│  通信协议 | 状态管理 | 事件总线               │
├─────────────────────────────────────────────┤
│            数据模拟层 (Mock Data)             │
│  模拟设备数据 | 模拟音频场景 | AI处理结果     │
│  性能指标数据 | 用户行为数据                  │
└─────────────────────────────────────────────┘
```

### 2.2 模块划分与职责

| 模块 | 职责 | 关键接口 |
|------|------|---------|
| **DeviceConnection** | 设备发现、蓝牙/WiFi配对、连接状态管理 | `scanDevices()`, `connect(deviceId)`, `on('DEVICE_CONNECTED')` |
| **DeviceMonitor** | 实时状态监控、电量/信号/温度、异常告警 | `getStatus()`, `on('BATTERY_LOW')`, `on('SIGNAL_WEAK')` |
| **AudioCapture** | 录音控制、波形动画、降噪模拟、清晰度评分 | `startRecord()`, `pause()`, `stop()`, `getWaveformData()` |
| **AIProcessing** | STT转写、翻译、摘要、置信度计算 | `process(audio, config)`, `on('TRANSCRIBE_PROGRESS')` |
| **ResultPresentation** | 结果格式化展示、交互操作、导出分享 | `renderResults(data)`, `export(format)`, `share()` |
| **PerformanceReport** | 性能数据收集、图表生成、体验分析 | `generateReport()`, `getMetrics()` |

### 2.3 通信协议设计

基于 JSON 的事件驱动通信：

```json
{
  "event": "DEVICE_CONNECTED",
  "timestamp": 1716700000000,
  "payload": {
    "device_id": "earbuds_001",
    "device_name": "Soundcore Liberty 4",
    "connection_type": "bluetooth",
    "battery": 85,
    "signal_strength": -45,
    "firmware_version": "v2.3.1"
  }
}
```

核心事件列表：
- `DEVICE_SCANNING` → `DEVICE_FOUND` → `DEVICE_CONNECTED` → `DEVICE_DISCONNECTED`
- `VOICE_STREAM_START` → `VOICE_DATA` → `VOICE_STREAM_END`
- `AI_PROCESSING_START` → `TRANSCRIBE_PROGRESS` → `AI_RESULT_READY`
- `BATTERY_LOW` / `SIGNAL_WEAK` / `FIRMWARE_UPDATE_NEEDED`

## 三、前端技术选型

### 3.1 推荐技术栈（纯前端单文件方案）

| 技术 | 用途 | 说明 |
|------|------|------|
| **HTML5 + CSS3** | 页面结构 | 使用 CSS Grid/Flexbox 布局，CSS 动画 |
| **Vanilla JavaScript (ES6+)** | 逻辑控制 | 无需框架，降低复杂度 |
| **Canvas API** | 录音波形可视化 | 实时绘制动态音频波形 |
| **Chart.js (CDN)** | 性能报告图表 | 柱状图、折线图、雷达图 |
| **CSS Animations/Transitions** | 过渡动画 | 连接进度动画、脉冲效果 |

### 3.2 为什么不选框架

- 交付物为 **HTML 单文件**，使用 React/Vue 需要构建工具链
- 模拟系统无需复杂状态管理，EventEmitter 模式足够
- 保持轻量、易于评委直接打开运行

## 四、关键交互实现要点

### 4.1 设备连接动画

```
实现方案：
1. 扫描动画：圆形雷达扩散效果（CSS radial-gradient + animation）
2. 设备列表：卡片式展示，点击连接
3. 连接过程：3秒模拟延迟，显示"配对中→认证中→已连接"三步进度
4. 连接成功：绿色脉冲动画 + Toast 提示
5. 连接失败：红色抖动动画 + "重新连接"按钮
```

### 4.2 录音波形

```
实现方案：
1. Canvas 绘制动态正弦波 + 随机扰动模拟真实波形
2. 音量条：实时变化的柱状图（0-100%）
3. 参数面板：采样率(48kHz)、位深(16bit)、降噪等级
4. 三按钮：开始(大圆红色) / 暂停(方形) / 停止(方形)
```

### 4.3 AI 处理流程动画

```
实现方案（串行分步展示）：
1. 阶段一：语音转文字 — 逐字出现的打字机效果 + 置信度条
2. 阶段二：翻译 — 原文/译文左右对照，翻译进度条
3. 阶段三：摘要 — 加载动画后出现摘要卡片
4. 每个阶段显示耗时、使用模型、置信度
```

### 4.4 场景模式切换

```
实现方案：
1. Tab 切换：会议/学习/办公/日常
2. 切换时界面元素变化：
   - 会议模式：强调多人转写、纪要生成
   - 学习模式：强调翻译对照、重点标注
   - 办公模式：强调语音指令、快速操作
   - 日常模式：强调摘要总结、时间线
3. 切换动画：淡入淡出过渡
```

### 4.5 结果展示

```
实现方案：
1. 对话气泡：按说话人分色，左/右气泡区分
2. 翻译卡片：原文在上、译文在下，可切换显示
3. 摘要卡片：要点列表式，支持折叠展开
4. 操作栏：导出(Word/TXT)、分享、修改翻译、清除数据
5. 可信度提示：每个转写段落旁显示置信度百分比
```

### 4.6 性能报告

```
实现方案：
1. 指标卡片：响应时间、准确率、AI耗时、操作次数
2. 图表区：
   - 折线图：各阶段响应时间趋势
   - 柱状图：不同场景准确率对比
   - 雷达图：综合性能评估
3. 体验分析：自动生成的文字建议（如"低噪环境识别最优"）
4. 优化建议：个性化提示卡片
```

## 九、与竞品的差异化建议

1. **全链路可视化**：竞品通常只展示结果，我们要将"连接→采集→处理→结果"全链路以动画形式展示
2. **拟真设备交互**：加入蓝牙扫描雷达、配对认证弹窗、信号强度动态条等拟真元素
3. **场景智能感知**：根据录音内容关键词自动切换处理模式（会议→纪要，外语→翻译）
4. **性能报告**：不仅展示数据，还提供体验瓶颈分析和个性化优化建议
5. **隐私透明**：录音前展示权限说明，提供"仅本地处理"选项和数据清除功能
